Layouts

Das normale Layoutverhalten in HTML ist das Flowlayout. Die Elemente werden in der Reihenfolge angezeigt, wie sie in der Dokumentenstruktur vorkommen. (also im Dokumentenfluss)
Von oben nach unten für Blockelemente und von links nach rechts für Inline-Elemente.

Wir haben bereits die position-Eigenschaft kennengelernt, über die die Positionierung außerhalb des Dokumentenflusses gesteuert werden kann. Daneben gibt es noch die float-Eigenschaft mit der die Positionierung eines Elements so erfolgen kann, dass Text- und Inline-Elemente um das Element herumfliessen.

Für komplexere Layouts wurde in der Vergangenheit oftmals das Tabellenlayout missbraucht. Spätestens jedoch bei responsiven Webseiten und semantischen Webseiten (a11y, Screenreader), stößt auch das Tabellenlayout an seine Grenzen.
Das w3c hat daher inzwischen 2 neue Layouts hinzugefügt, die inzwischen von jedem Browser unterstützt werden:

  • Flexbox
  • Grid

Flexbox

Mit Flexbox lassen sich Elemente vertikal oder horizontal anordnen. Reicht der Platz in der Horizontalen oder Vertikalen nicht aus, können die Elemente auf mehrere Reihen aufgeteilt werden.
Das besondere des Flexlayouts ist, dass das Aussehen von der Größe der Elemente abhängig ist. Man könnte sagen, es ist ein bottom-up-Layout. Es eignet sich daher hervorragend für responsive Webseiten, bei denen der Content sehr variabel ist. 

flex-container {
  display: flex;
  flex-direction: row/column/row-reverse/column-reverse;
  flex-wrap: wrap/nowrap;
  gap: n;
  justify-content: flex-start/flex-end/center/space-between/space-around;
  align-content: flex-start/flex-end/center/space-between/space-around;
  align-items: stretch/center/flex-start/flex-end/baseline/;
}
flex-item {
  flex-grow: n;
  flex-shrink: n;
  flex-basis: n;
  flex: grow shrink basis;
  align-self: center/stretch/flex-start/flex-end;
}

Grid

Beim Gridlayout wird die Anordnung der Elemente in einem Gridsystem festgelegt. Im Gegensatz zum Flexbox-Layout hat die Größe der Elemente keinen Einfluss auf das Aussehen. Stattdessen wird den Elementen der verfügbare Platz innerhalb des Grids zugewiesen. Man könnte sagen, dass es ein Top-Down-Layout ist. 

grid-container {
  display: grid/inline-grid;
  grid-template-columns: auto auto auto; (3 Spalten)
  grid-template-columns: 80px 200px auto 40px; (4 Spalten)
  grid-template-columns: 80px 200px auto 40px; (4 Spalten)
  grid-template-rows: 80px auto;
  grid-template-areas: 
      'header header header'
      'menu main main'
      'menu footer footer';
  justify-content: start/end/center/space-around/space-between/space-evenly; (falls Container breiter als das eigentliche Grid)
  align-content: start/end/center/space-around/space-between/space-evenly; (falls Container höher als das eigentliche Grid)
}
grid-item {
  grid-column: 'colstart / colend' 'colstart / span n';
  grid-row: 'rowstart / rowend' 'rowstart / span n';
  grid-area: 'grid-row-start / grid-column-start / grid-row-end / grid-column-end';
  grid-area: areaname;
}
